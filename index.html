OAuth 2.0

Antonio Gámiz

¿Por qué OAuth 2.0?

  • Seguridad de APIs
  • Intercambio de contraseñas
  • Delegación de autoridad
  • autenticación
  • autorización
  • identificar a las partes involucradas

¿Qué es OAuth 2.0?

Estándar que permite a las aplicaciones un acceso limitado (scopes) a los datos de los usuarios, sin tener que proporcionar las credenciales de dicho usuario, desacoplando la autenticación y la autorización a los datos.

Tarjeta de acceso a un hotel

Glosario

  • autentificación !== autorización
  • Resource owner (el usuario)
  • Resource server (la API)
  • Client
  • Authorization server
  • First party / Third party client
App de Spotify para iOS / Iniciar sesión con Google

Tipos de Grant

  • Authorization code grant
  • Implicit grant
  • Resource owner credentials grant
  • Client credentials grant
